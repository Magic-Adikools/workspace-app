🔏 Polityka prywatności
Ostatnia aktualizacja: czerwiec 2025
1. Informacje ogólne
Workspace to osobista aplikacja produktywności służąca do zarządzania zadaniami, zakupami, projektami i notatkami. Aplikacja jest tworzona z poszanowaniem prywatności użytkownika.
2. Gromadzone dane
Aplikacja gromadzi wyłącznie dane wprowadzone przez użytkownika:
  • Treść zadań, projektów, notatek i list zakupów
  • Ustawienia aplikacji (motyw, język, rozmiar czcionki)
  • Dane konta Google (imię, adres e-mail, zdjęcie profilowe) — wyłącznie po zalogowaniu
Aplikacja nie gromadzi danych o lokalizacji, urządzeniu, ani zachowaniu użytkownika.
3. Przechowywanie danych
Dane są przechowywane na dwa sposoby:
  • Lokalnie — w pamięci przeglądarki (localStorage) na urządzeniu użytkownika, bez wysyłania ich gdziekolwiek
  • W chmurze — po zalogowaniu przez Google, dane są synchronizowane z bazą Firebase Firestore należącą do Google LLC, zabezpieczoną zgodnie z polityką Google
4. Logowanie przez Google
Aplikacja umożliwia opcjonalne logowanie przez konto Google (Google Sign-In). Logowanie jest dobrowolne — aplikacja działa w pełni bez konta. Po zalogowaniu:
  • Dane są synchronizowane między urządzeniami przez Firebase
  • Aplikacja przechowuje token sesji Google w pamięci przeglądarki
  • Dane konta (imię, e-mail) są wyświetlane tylko w interfejsie aplikacji
Logowanie Google podlega Polityce prywatności Google.
5. Udostępnianie danych
Dane użytkownika nie są sprzedawane, udostępniane ani przekazywane żadnym podmiotom trzecim, za wyjątkiem infrastruktury Google Firebase niezbędnej do synchronizacji (gdy użytkownik jest zalogowany).
6. Prawa użytkownika
Użytkownik ma prawo do:
  • Dostępu do swoich danych — są widoczne bezpośrednio w aplikacji
  • Usunięcia danych — przez wyczyszczenie danych przeglądarki lub usunięcie konta Firebase
  • Eksportu danych — przez funkcję eksportu JSON dostępną w ustawieniach
  • Rezygnacji z synchronizacji — przez wylogowanie z konta Google
7. Bezpieczeństwo
Dane w chmurze są zabezpieczone regułami bezpieczeństwa Firebase — każdy użytkownik ma dostęp wyłącznie do swoich danych. PIN blokady jest przechowywany lokalnie jako hash — nigdy jako czysty tekst.
8. Kontakt
W razie pytań dotyczących prywatności, skontaktuj się przez ustawienia aplikacji lub bezpośrednio z deweloperem.
Last updated: June 2025
1. General Information
Workspace is a personal productivity application for managing tasks, shopping lists, projects and notes. The application is built with respect for user privacy.
2. Data Collected
The application collects only data entered by the user:
  • Content of tasks, projects, notes and shopping lists
  • Application settings (theme, language, font size)
  • Google account data (name, email address, profile picture) — only when signed in
The application does not collect location data, device information or user behaviour data.
3. Data Storage
Data is stored in two ways:
  • Locally — in the browser's localStorage on the user's device, without sending it anywhere
  • In the cloud — when signed in via Google, data is synchronised with Firebase Firestore, owned by Google LLC and secured according to Google's policies
4. Google Sign-In
The application offers optional sign-in via Google account. Sign-in is voluntary — the application works fully without an account. When signed in:
  • Data is synchronised across devices via Firebase
  • The application stores the Google session token in browser storage
  • Account data (name, email) is displayed only within the application interface
Google Sign-In is subject to Google's Privacy Policy.
5. Data Sharing
User data is never sold, shared or transferred to any third parties, except for Google Firebase infrastructure required for synchronisation (when the user is signed in).
6. User Rights
The user has the right to:
  • Access their data — visible directly in the application
  • Delete their data — by clearing browser data or deleting the Firebase account
  • Export their data — via the JSON export function in settings
  • Opt out of synchronisation — by signing out of the Google account
7. Security
Cloud data is secured by Firebase security rules — each user has access only to their own data. The lock PIN is stored locally as a hash — never as plain text.
8. Contact
For any privacy-related questions, please contact via the application settings or directly with the developer.
